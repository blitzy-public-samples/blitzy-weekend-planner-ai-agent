## Screenshot Capture Instructions

Capture a screenshot of the Weekend Planner application's input form showing validation errors. The primary validation scenario to capture is when the End Date is set before the Start Date, which triggers an inline error message.

**Key Requirements:**
- The form should display the inline validation error
- The "Generate Plan" button must be in the disabled state
- The error message should be clearly visible and readable

---

## How to Trigger Validation Error

Follow these steps to reproduce the validation error state for capture:

1. Open the Weekend Planner application in your browser at `http://localhost:5173`
2. Enter a valid location in the Location field (e.g., "San Francisco")
3. Select a Start Date using the date picker (e.g., select a Saturday like December 14, 2024)
4. Select an End Date that is BEFORE the Start Date (e.g., select Friday December 13, 2024)
5. The inline error message "End date must be on or after start date" should now appear
6. The "Generate Plan" button should automatically be disabled

**Note:** Do not fill in optional fields (Kids Ages, Preferences) unless they help illustrate the validation state more clearly.
